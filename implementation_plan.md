# 프로젝트 종합 진단 및 개선 제안 (Reconnection AI)

현재 프로젝트는 "코스믹 다크 테마"의 미니멀하면서도 프리미엄한 UI, 그리고 병렬 처리 기반의 고도화된 프롬프트를 갖춘 **매우 훌륭한 상태**입니다. 

하지만 **실제 사용자가 들어오고 트래픽이 발생하기 전(Production Ready)**에 보완하면 좋을 몇 가지 잠재적 불안 요소와 UX 디테일이 있습니다. 사용성 및 안정성 검토 결과를 아래와 같이 제안합니다.

---

## 1. 🚨 데이터 안정성: 브라우저 용량(Storage) 한계 돌파
지금까지 개발된 구조는 사용자와의 모든 분석 기록(각 5,000자 이상)을 `Zustand persist`를 통해 `localStorage`에 저장하고 있습니다.

- **위험 요소**: 브라우저 `localStorage`는 최대 **5MB**까지만 저장 가능합니다. 고품질 텍스트 리포트가 10~20개 이상 누적되면, 용량 초과 에러가 발생하며 페이지가 먹통이 될 수 있습니다.
- **해결 제안**: `localStorage` 대신 용량 제한이 사실상 없는 **웹 IndexedDB**(`idb-keyval` 라이브러리 사용)로 Zustand 스토리지를 교체해야 합니다. 

## 2. 🔔 UX 고도화: Alert 제거 및 Toast 알림 도입 (Global Rule 준수)
- **위험 요소**: `useSajuStore`, `analysis/page.tsx` 등에서 비동기 로딩 에러 시 `alert("오류가 발생했습니다")`를 사용 중입니다. 이는 모바일 환경에서 매우 투박하고 흐름을 깹니다. (사용자가 요청한 코딩 규칙 누락)
- **해결 제안**: `react-hot-toast`와 같은 가벼운 알람 라이브러리를 설치해, 화면 상단/하단에 부드럽게 나타나는 **스낵바(Toast) 알림**으로 교체해야 합니다.

## 3. 🤖 AI 서버 안정성: 자동 재시도(Retry) 로직
- **위험 요소**: Gemini AI가 순간적인 트래픽 지연이나 토큰 제한에 걸려 `fetch`를 실패하면 사용자는 아무 정보도 얻지 못합니다.
- **해결 제안**: `api/reunion/route.ts` 등에 실패 시 자동으로 **1~2회 재요청(Retry)**하는 방어 로직을 씌워 응답 성공률을 99%까지 끌어올립니다.

## 4. 🔗 마케팅 & 바이럴: SEO/오픈그래프(OG) 메타데이터 보강
- **위험 요소**: 현재 지인이나 카카오톡 대화방에 서비스 링크(URL)를 복사해서 공유할 때, 썸네일 이미지나 매력적인 설명 피드가 노출되지 않습니다.
- **해결 제안**: `layout.tsx`에 `openGraph` 및 `twitter` 메타 태그를 완벽하게 세팅하여 링크 클릭률을 높여야 합니다.

## 5. 💳 결제 모듈화 준비
- **위험 요소**: 현재 Standard(1,900원) 업그레이드 버튼이 클릭 시 무료로 직접 API를 태우고 있습니다.
- **해결 제안**: 당장 실결제를 연동하지 않더라도, 버튼 클릭 시 "가상 결제 프로세스 연동 대기중" 같은 UI 처리나 컴포넌트화를 통해 실제 PG사(토스, 포트원 등)를 꽂기 쉬운 구조로 분리해두면 유지보수가 편해집니다.

---

> [!IMPORTANT]
> **유저 피드백 요청**
> 위 1~5번 항목 중 1번(IndexedDB 도입)과 5번(결제 모듈화 도입) 외 대부분의 기능은 구현 완료되었습니다.

# Phase 2: 랜딩 페이지 UI/UX 극대화 (진행 예정)

메인 페이지의 콘텐츠 길이를 늘리고, 사용자의스크롤을 유도하기 위해 다음 3가지 섹션을 도입합니다.

## 조화로운 3가지 섹션 구성안

### 1단계: "이런 분들을 위해 만들었습니다" (공감 체크리스트)
- **위치**: 히어로 배너(최상단) 직후에 배치. 사용자가 처음 스크롤을 내리자마자 뼈아픈 공감 리스트를 보여줍니다.
- **디자인 원칙**: 코스믹 톤 앤 매너에 맞게 반투명한 체크박스(`glass-card`)들과 흔들리는 애니메이션을 배치하여 시선을 끕니다.

### 2단계: 소셜 프루프 (무한 롤링 및 카운터)
- **위치**: 중간 섹션 (고객 만족도나 리뷰 영역 근처).
- **디자인 원칙**: Framer Motion 또는 Tailwind CSS의 `@keyframes`를 활용해 리뷰 카드들이 왼쪽으로 끊임없이 스무스하게 넘어가는 무한 캐러셀(Carousel)을 구현합니다. 또한 상단에 "지금까지 N개의 꼬인 인연을 분석했습니다"라는 텍스트를 배치합니다.

### 3단계: Premium 리포트 엿보기 (실제 컴포넌트 목업)
- **위치**: 무한 롤링 리뷰 섹션 바로 아래, 결제 유도 직전.
- **디자인 원칙**: 실제 제공되는 `GoldenWindowTimeline` 및 `MonthlyEnergyFlow` UI의 디자인 일부분을 흐림 처리(Blur)와 모자이크로 배치하여 "이런 기깔나는 로드맵을 확인할 수 있습니다"라는 느낌을 자아냅니다.

## Verification Plan

### 수동 검증 (Manual Verification)
- 모바일(가로폭 360~480px) 환경에서 무한 스크롤이 자연스럽게 작동하는지 확인.
- Framer Motion / Tailwind 애니메이션 성능이 저하되지 않는지 확인.
- 전체 스크롤 레이아웃 흐름이 자연스러운지 확인.
